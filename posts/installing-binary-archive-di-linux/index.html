Install File Binary Archive Zip Tar Gz Di Linux

2024-10-12 13:52:03 +0700 WIB

Sebenernya orang-orang umumnya tidak peduli kamu menggunakan linux, bahkan orangtua kamu. Tapi sebagian lagi orang non umum peduli, khususnya dalam dunia teknologi.

Saya mengakui dalam kebutuhan tertentu, macOS dan Windows adalah pilihan. Tapi dalam dunia teknologi linux superior, tapi kita sedang tidak membandingkan OS disini.

Karena akhir-akhir ini lebih sering bekerja di server, linux jauh lebih stabil ketimbang windows, lalu setelahnya BSD Based.

Khususnya dalam kebutuhan installasi program sudah ada package manager, yang mempermudah installasi aplikasi sekaligus library di dalamnya.

Cuman ada beberapa kasus aplikasi yang tidak tersedia di dalam package manager, atau hanya support di snap kalau di ubuntu, atau hanya di AUR kalau di ArchLinux.

Berikut adalah proses intallasi Aplikasi dari binary archived file, Contoh kasus Intallasi ZIG.

Ketika awal membuka terminal umumnya akan berada di home path (~), anda bisa download file menggunakan

wget https://ziglang.org/download/0.13.0/zig-linux-x86_64-0.13.0.tar.xz

Selanjutnya extract file .tar.xz, perintah bisa di sesuaikan dengan format archived.

tar xvf zig-linux-x86_64-0.13.0.tar.xz

Ubah nama folder biar gampang di akses dan di ingat.

mv zig-linux-x86_64-0.13.0/ zig/

pindah berkas installasi ke /usr/local/ atau bisa juga menggunakan direktori standar lain,

mv zig/ /usr/local/

Tambah folder tadi ke dalam environment PATH

export PATH=/usr/local/zig:$PATH

Testing binary, harusnya sudah bisa ketika mengetik perintah zig

zig
info: Usage: zig [command] [options]
...

kalau sudah aman, bisa nemabahkan perintah export ke dalam .bashrc atau .bash_profile di dalam folder home jika menggunakan BASH, dan bisa di tumpuk dengan binary yang lain.

export PATH=$PATH:/usr/local/go/bin:/usr/local/hugo/bin:/usr/local/zig

Kenapa adanya arikel ini? Sebelumnya sih tinggal copas dari internet untuk proses intallasinya, cuman karena standar tutorial satu dengan yang lain berbeda khususnya dalam pemilihan path installasi, ada yang di dalam /opt/ ada yang juga di .local walau semua tergantung kebutuhan, tapi tetap perlu standarisasi.
